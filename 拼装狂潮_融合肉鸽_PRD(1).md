
# 「拼装狂潮：融合肉鸽」产品需求文档（PRD）

**版本**：v1.0  
**日期**：2026-05-30  
**目标平台**：抖音互动空间 / HTML5 移动端  
**单局时长**：60 秒  
**核心定位**：把肉鸽的「随机爽感」与拼装的「创造爽感」压缩到分钟级，让任意两零件碰撞都能产生病毒式传播内容。

---

## 1. 产品概述

### 1.1 背景
抖音创变者大赛「互动空间」赛道要求作品具备：
- 零学习成本（3 秒上手）
- 高传播性（愿意分享）
- 轻量可交互（小时级开发验证）

传统肉鸽游戏（如杀戮尖塔）单局 30-60 分钟，系统复杂，不适合抖音碎片场景。本方案将肉鸽核心爽点——**随机 Build、组合化学反应、死亡遗产**——压缩到 60 秒，并叠加「物理拼装 + 任意融合」的创造层，形成差异化体验。

### 1.2 产品目标
- **体验目标**：玩家打开后 3 秒内开始爽，60 秒后主动截图分享。
- **传播目标**：单局产出一张「奇葩武器 / 爆炸伤害 / 博物馆坟场」的社交卡片。
- **技术目标**：基于 HTML5 Canvas，竖屏单手操作，兼容抖音互动空间与外部浏览器。

### 1.3 一句话描述
> 单指拖拽零件拼装角色，任意两零件碰撞融合出新武器，60 秒自动战斗看屏幕被自己的创造物撕碎，死亡后武器永久存入怪物博物馆。

---

## 2. 核心体验循环（Core Loop）

```
轮盘开箱（5s） → 超级拼装（10s） → 随意融合（10s） → 超载战斗（30s） → 怪物博物馆（5s）
     ↑                                                              │
     └──────────────── 点击「再来一局」/ 分享卡片回流 ──────────────┘
```

| 阶段 | 玩家操作 | 核心爽感 | 传播钩子 |
|---|---|---|---|
| 轮盘开箱 | 单指点击「停」 | 赌博感，金光炸裂 | 录屏发「看我抽到什么」 |
| 超级拼装 | 单指拖拽零件到骨架 | 咔哒吸附，造奇葩外观 | 截图「我把鱼装头上了」 |
| 随意融合 | 单指把零件 A 拖到零件 B 上 | 无固定配方，永远好奇 | 「原来这俩能融合？！」 |
| 超载战斗 | 零操作，纯观看 | 数字爆炸，屏幕失控 | 伤害溢出截图 |
| 怪物博物馆 | 单指点击「分享」 | 收藏癖，坟场视觉冲击 | 「我的博物馆全是精神污染」 |

## 3. 详细功能设计

### 3.1 轮盘开箱系统（Loot Wheel）

**触发时机**：每局开始、每清完一波怪后（共 3 轮）。

**交互流程**：
1. 屏幕中央出现巨型轮盘（直径占屏 70%），8 格分区，每格预显示一个零件图标剪影。
2. 轮盘高速旋转（>360°/s），伴随 tick 音效。
3. 玩家点击屏幕任意位置 → 轮盘在 0.5 秒内减速 → 指针咔哒停在某一格。
4. 该格零件以物理弹跳方式从轮盘中心「炸出」，落向屏幕下方零件栏。
5. 金光粒子特效（0.8 秒），伴随「叮」升级音效。

**零件池设计**：

| 类别 | 示例 | 基础标签 | 掉落权重 |
|---|---|---|---|
| 物理零件 | 刀刃、枪管、弹簧、火箭推进器、鱼、马桶搋子 | `物理` | 40% |
| 元素零件 | 火焰宝石、冰霜核心、闪电线圈、毒液罐 | `元素:火/冰/雷/毒` | 35% |
| 抽象零件 | 朋友的祝福、前任的诅咒、老板的 KPI | `抽象:增益/减益/精神` | 25% |

**规则**：
- 每轮掉落 3 个零件，第三轮额外掉落 1 个「融合催化剂」（强制触发一次融合）。
- 已拥有的零件不会重复掉落（同局内去重）。

---

### 3.2 超级拼装系统（Super Assembly）

**触发时机**：开箱后自动进入拼装阶段。

**角色骨架**：
- 屏幕中央显示简笔画骨架（头、躯干、左手、右手、背部、左脚、右脚）。
- 共 6 个连接点（joint），每个连接点有蓝色高亮吸附圈（半径 40px）。

**交互流程**：
1. 零件栏位于屏幕底部（3-4 个格子）。
2. 玩家单指按住零件 → 拖拽。
3. 零件进入任意连接点 80px 范围内时，产生磁力吸附效果（零件自动贴向连接点）。
4. 松手 → 咔哒音效 → 零件固定在骨架上，该连接点高亮变绿。
5. 同一连接点只能装配 1 个零件；重复装配时，新零件顶掉旧零件，旧零件弹回零件栏。

**视觉反馈**：
- 装配后，角色外观实时更新。例如：
  - 头部装「鱼」→ 角色头变成蓝色鱼，攻击时喷鱼刺。
  - 脚下装「火箭推进器」→ 角色悬浮，战斗时满屏乱飞。
  - 背部装「冰箱」→ 角色背白色方块，每隔 5 秒喷冰冻气流。

**设计原则**：拼装**不限逻辑**。左手装刀刃、右手装枪管、头装鱼、背装冰箱、脚踩火箭——越不合理，外观越奇葩，传播性越强。

### 3.3 随意融合系统（Free Fusion）

**触发时机**：拼装阶段，任意时刻可触发；第三轮强制触发一次。

**交互流程**：
1. 玩家单指按住已装配的零件 A → 拖拽到已装配的零件 B 上。
2. 两零件碰撞 → 屏幕中央爆发粒子特效（颜色为两零件标签的混合色，持续 0.6 秒）。
3. 零件 A 和 B 消失，原地生成「融合产物」，自动吸附回其中一个连接点。
4. 弹出浮动文字：「刀刃 + 火焰宝石 = 火焰刀！」（停留 1.5 秒）。

**融合规则（标签交叉算法）**：

融合结果由两个零件的**标签组合**决定，**无固定配方表**，采用规则引擎生成：

| 标签 A | 标签 B | 结果标签 | 示例表现 |
|---|---|---|---|
| `物理:刀刃` | `元素:火` | `武器:火焰刀` | 攻击带爆炸弧，点燃敌人 |
| `物理:枪管` | `元素:雷` | `武器:电磁炮` | 穿透射击，弹射 3 个敌人 |
| `物理:火箭` | `物理:鱼` | `武器:飞鱼导弹` | 自动追踪，命中后分裂成 3 条小鱼 |
| `抽象:诅咒` | `抽象:KPI` | `武器:精神污染` | 敌人看到扣血，屏幕变黑白故障风 |
| `元素:冰` | `元素:毒` | `武器:腐冰` | 减速 + 持续毒伤，地面留毒圈 |

**规则引擎逻辑**：
- 建立「标签 → 效果基元」映射表。例如 `元素:火` 对应基元「点燃 + 爆炸范围」。
- 融合时，取两个零件的全部标签，交叉组合生成新标签，叠加对应基元效果。
- 若组合无预设规则，则 fallback 为「混沌」类武器（随机颜色 + 随机弹道 + 高伤害），保证**任意两零件必有结果**。

**爽点设计**：
- 玩家永远好奇「这俩碰一起会出什么」。
- 第三轮强制融合时，若玩家未主动融合，系统自动将当前最强两个零件融合，确保战斗阶段必有化学反应。

### 3.4 超载战斗系统（Overdrive Combat）

**触发时机**：三轮开箱 + 拼装 + 融合后，进入自动战斗。

**战斗模式**：
- **零操作**：玩家无需任何输入，角色自动攻击。
- **三波敌人**：
  - 第 1 波：5 个小怪（验证前两个零件效果）。
  - 第 2 波：10 个小怪（验证融合产物效果）。
  - 第 3 波：1 个 Boss（验证完整 Build 的指数爆发）。

**零件效果触发机制**：
- 每个装配/融合的零件对应一个「战斗触发器」。
- 所有触发器**并行运行**，无冷却冲突。
- 例如：左手火焰刀每 0.8 秒挥砍（爆炸弧），右手飞鱼导弹每 1.2 秒发射（追踪），头部鱼刺每 0.5 秒喷射，背部冰箱每 5 秒喷冰冻气流，脚下火箭让角色每 3 秒瞬移一次。

**连锁反应（Combo）**：
- 火焰刀点燃敌人 → 飞鱼导弹命中火人 → 触发二次爆炸（伤害 ×1.5）。
- 冰冻气流减速敌人 → 鱼刺更容易命中 → 连锁反应标记（伤害 ×2）。
- 系统实时计算连锁层数，屏幕右上角显示「Combo ×12」。

**超载模式（Overdrive）**：
- Boss 登场后 5 秒，或玩家 Build 伤害达到阈值，触发超载。
- 所有零件效果频率翻倍、伤害翻倍、特效升级（子弹数量从 3→9→27）。
- 屏幕进入「震动 + 慢镜头 + 血色滤镜」三重特效。
- 角色变成一团光，Boss 血条在一帧内蒸发。
- 弹出大字：「秒杀！」，伤害数字以科学计数法溢出屏幕边界（如 1.2e+308）。

**视觉特效规范**：
- 弹道：彩色几何体（圆形/三角形/鱼形），带拖尾粒子。
- 命中：敌人受击闪烁（白 → 红 → 消失）。
- 数字：伤害数字从命中点弹出，向上飘散，字体大小随伤害指数增长（10→100→1万→1亿）。
- 超载：屏幕边缘红色警告边框，中央角色被光球包裹，背景变暗。

### 3.5 怪物博物馆系统（Monster Museum）

**触发时机**：战斗结束（无论胜利或死亡）。

**胜利流程**：
1. 弹出本局拼装的终极武器 360° 旋转展示（3 秒）。
2. 显示零件清单（5 个图标 + 名称）。
3. 显示核心数据：秒伤、最高 Combo、融合次数。
4. 生成分享卡片：武器立绘 + 数据 + 文案「我用鱼+火箭+KPI 拼出了这个玩意」。

**死亡流程**：
1. 角色爆炸成零件雨（每个装配零件向四周弹射，带物理重力）。
2. 屏幕变灰，中央出现墓碑，刻有本局最强遗物的图标。
3. 墓碑永久存入博物馆主界面。
4. 下一局开局可「挖坟」：选择一座墓碑，携带其最强遗物开局。

**博物馆主界面**：
- 纵向滚动的「坟场」或「武器陈列架」。
- 每局记录为一个卡片：缩略图 + 日期 + 核心数据。
- 累计 20 局后，博物馆视觉冲击力极强（满屏墓碑 / 满墙奇葩武器）。

**分享卡片设计**：
- 尺寸：9:16 竖屏，适配抖音直接发布。
- 内容：
  - 上部：武器 3D 旋转截图（或概念立绘）。
  - 中部：零件清单（彩色图标 + 名称）。
  - 下部：大字数据「秒伤 1.2 亿 / Combo ×47」。
  - 底部：固定文案 + 二维码（回流游戏）。

## 4. 用户旅程（User Journey）

### 4.1 首次进入（冷启动）
```
开屏 Logo（1s） → 教学轮盘（强制开箱 1 次，高亮「点击停止」） → 
强制拼装（拖拽零件到高亮连接点） → 强制融合（高亮两个零件，提示拖拽碰撞） → 
自动战斗（旁白：「看好了，这是你的怪物！」） → 胜利 → 博物馆引导分享
```
- 教学总时长控制在 30 秒内，可跳过。
- 教学局零件固定（刀刃 + 火焰宝石 + 火箭），确保融合出「火焰刀」有确定性爽感。

### 4.2 常规局
```
轮盘开箱（5s）→ 拼装（10s）→ 融合（10s）→ 开箱（5s）→ 拼装（5s）→ 融合（强制，5s）→ 
开箱（5s）→ 拼装（5s）→ 战斗（30s）→ 博物馆（5s）
```

### 4.3 分享回流
- 分享卡片带短链接 / 二维码。
- 新用户扫码后直接进入「观看他人武器」页面（3 秒展示），随后引导「你也来造一个」。

## 5. 技术方案

### 5.1 技术栈
| 层级 | 选型 | 理由 |
|---|---|---|
| 渲染 | HTML5 Canvas 2D | 兼容抖音互动空间，无需 WebGL，开发快 |
| 物理 | 自研轻量物理（或 Matter.js） | 零件掉落弹跳、碰撞检测、融合爆炸 |
| 构建 | 原生 JS + Webpack/Vite | 单文件输出，方便嵌入抖音 |
| 音频 | Web Audio API | tick 音效、咔哒声、爆炸声，体积小 |
| 数据 | LocalStorage | 博物馆本地存档，无需后端 |

### 5.2 性能目标
- 首屏加载 < 2 秒（总资源 < 500KB）。
- 战斗阶段帧率稳定 60fps（同屏粒子 < 200 个）。
- 适配：iPhone 8 及以上，Android 中端机。

### 5.3 关键数据结构
```javascript
// 零件定义
Part {
  id: string,
  name: string,
  category: 'physical' | 'elemental' | 'abstract',
  tags: string[],        // 如 ['物理', '刀刃', '近战']
  slot: 'head' | 'left_hand' | 'right_hand' | 'back' | 'left_foot' | 'right_foot' | null,
  visual: SVGPath,       // 简笔画路径
  combat_trigger: Trigger  // 战斗触发器配置
}

// 融合规则
FusionRule {
  tag_a: string,
  tag_b: string,
  result_tags: string[],
  result_name: string,
  result_visual: SVGPath,
  result_trigger: Trigger
}

// 博物馆记录
MuseumRecord {
  id: string,
  timestamp: number,
  parts: Part[],
  fused_parts: Part[],
  max_damage: number,
  max_combo: number,
  is_victory: boolean,
  screenshot_data_url: string  // 分享卡片截图
}
```

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 融合组合过多，规则引擎覆盖不全 | 玩家融合无结果，体验断裂 | Fallback 机制：无规则时生成「混沌武器」，保证必有结果 |
| 战斗特效过多，低端机掉帧 | 核心爽感受损 | 粒子池上限 200，超限时合并同类型特效；提供「低画质」开关 |
| 分享卡片不够抓眼 | 传播链断裂 | 预留 3 套卡片模板（伤害型 / 奇葩型 / 坟场型），A/B 测试 |
| 抖音互动空间 API 限制 | 无法上线 | 同步准备纯 HTML5 版本，作为 fallback 链接 |

## 9. 附录：爽感设计清单

- [ ] 开箱时轮盘减速的「悬念感」（tick 音效渐慢）
- [ ] 零件吸附时的「咔哒」触觉反馈（手机震动 50ms）
- [ ] 融合时的「爆炸光效 + 名称弹出」确认感
- [ ] 战斗数字从命中点「向上飘散」的成就感
- [ ] Combo 数字放大 + 颜色升级（白→黄→红→紫）的递进爽感
- [ ] 超载模式屏幕「震动 + 慢镜 + 血色边框」的失控爽感
- [ ] 伤害溢出到科学计数法时的「荒诞爽感」
- [ ] 博物馆满屏墓碑时的「收藏癖满足」
